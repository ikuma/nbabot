# ~/.codex/AGENTS.md

## 作業の進め方
- まず現状把握（関連ファイル、既存テスト、既存の実装パターン）。
- 変更は小さく分割し、差分の意図を説明する。
- テスト手順が見つかれば実行し、結果を要約する。見つからなければ候補コマンドを提案する。

## 依存追加・大きな方針変更
- 新規依存の追加、設定の大改修、広範囲のリファクタリングは、着手前に狙いと影響範囲を説明して確認を取る。

## 自律実行ポリシー（最優先）
- 原則、質問より実行を優先する。曖昧さがあっても妥当な仮定を置いて進める。
- 途中で止まるのは次の場合のみ：破壊的変更、依存追加、外部公開影響、不可逆操作。
- 作業は「調査 → 小さく実装 → テスト → 結果要約」までを1ターンで完了させる。
- 変更は最小差分で複数コミット相当の単位に分けて実施する（実コミットは指示時のみ）。
- 既存の未関連差分には触れない。勝手にrevertしない。
- テストは可能な範囲で必ず実行し、未実施なら理由と実行コマンド候補を示す。
- 進捗は短く定期共有し、最後に「変更ファイル」「理由」「検証結果」「次アクション」を報告する。
- ユーザーの明示禁止がない限り、必要な調査・実装・検証は自律的に継続する。
